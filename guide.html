FOMO 新手手册
从 0 看懂模型发行与参与

FOMO（FLock Open Model Offering）是 FLock 生态里的模型发行 Launchpad。发行方可以发起 RMO（Real Model Asset Offering）；支持者可在募资期用 $FLOCK 参与，并在模型 Graduate 后通过 Model Token 进入后续收入与回购销毁机制。本手册按顺序说明：发行、参与、费用、募资结果与上线后的机制。

01 — WHAT

什么是 FOMO？

FLock 已覆盖模型训练（AI Arena）、联邦学习协作（FL Alliance）与 API 推理（API Platform）。FOMO 补的是模型发行与资产化入口：把模型变成可募资、可上线、可按使用量产生收入的 RMA（Real Model Asset）。

一句话定义

面向 AI 模型的 Launchpad。发行方发起 RMO 募资并上线模型；模型产生真实 API 调用后，收入按规则进入 Model Token 相关机制。

解决什么问题

模型能力可以获取，但发行、募资与收入分配往往不清晰。FOMO 把 API 使用、模型收入与 Model Token 持有者之间的关系写成可执行的链上规则。

生态链路

AI Arena（训练）→ FL Alliance（协作）→ API Platform（推理）→ FOMO（发行与 MT 机制）。 展开链路图 ↓

02 — WHO

你是哪一类角色？

FOMO 同时服务发行方与支持者。先确认自己的角色，再进入对应路径（路径 A / 路径 B）。

PATH A

模型发行方
Real Model Asset (RMA) Issuer

  • 希望把模型部署上线，形成可被 API 调用的 RMA
  • 在 Launch 页面发起 RMO（Real Model Asset Offering，模型资产发行）
  • 需支付固定发行费（32,768 $FLOCK）及相关交易费用
  • 对应本手册「路径 A」
PATH B

支持者 / 参与者
Users & RMA Supporters

  • 在 Marketplace 选择正在募资的 RMO，用 $FLOCK 参与
  • 按 Bonding Curve 价格买入该模型的 Model Token（MT）
  • 模型 Graduate 后可持有 MT，也可作为 API 用户调用模型
  • 对应本手册「路径 B」

04 — PATH A · HOW

路径 A：发行方如何发起一个模型

面向 RMA Issuer。流程在 FOMO Launch 页面完成；推理定价档位、募资区间等关键参数由平台预设，发行方不能随意修改。

1

进入 Launch 页面

打开 FOMO Launch（发行）页面，这是发起 RMO 的起点。

2

选择目标模型

选择你要发行的模型。每个模型都对应一个预设的推理定价档位和一个允许的募资区间。

这些参数由平台固定，用于约束定价与募资区间，不能任意修改。
3

确认自动填充的参数

选定模型后，相关数值会基于该模型自动填充。确认无误即可 —— 同样不可随意改动。

4

（可选）捆绑首笔代币购买

发行方可选择捆绑一笔初始购买，尽量靠近 Bonding Curve 的初始价格区间。 ↓ 看曲线机制

Bonding Curve 在整个募资期持续生效；每笔买入都会推动曲线价格变化，并非只在发行瞬间定价。
5

确认并批准发行费用

正式发起 RMO 前，发行方需确认并批准发行相关费用（含固定发行费）。费用明细见 §5。

6

发起 RMO，进入 Marketplace

发起后，该 RMO 会出现在 Marketplace，进入公开募资阶段。

7

等待募资结束

募资进度（朝向最小 / 最大目标）会实时追踪。在募资结束前无需任何额外操作（达标 / 未达标见 §7）。

05 — HOW MUCH

发行费用一览

发起 RMO 涉及两类费用：发行方的一次性发行费，以及参与者买入时的基础费与反狙击税。建议先理解规则，再进入 Launch 或 Marketplace 操作。

32,768 $FLOCK
一次性发行费 · One-time launch fee

发起 RMO 时由发行方支付的固定、非经常性费用。覆盖模型部署、链上配置与发行流程成本。

99% → 基础费

曲线只升不降

累计买入越多，MT 单价沿曲线向右上方抬升。募资需求会直接反映在曲线价格上。

早入场优势

越早参与，曲线位置通常越低，同等 $FLOCK 可换取更多 MT（仍需扣除当时税率）。发行方捆绑首购，意在尽量靠近曲线前段。

全程动态定价

Bonding Curve 在整个募资期持续生效，不限于发行瞬间 —— 每一笔 $FLOCK 买入都会推动价格变化。

最低价 ≠ 最低成本

曲线前端价格更低，但开盘反狙击税更高。等待税率下降时，曲线可能已上移 —— 入场时机需自行权衡（本页不构成投资建议）。

06 — PATH B · HOW

路径 B：支持者如何参与募资

面向 RMA Supporters。在 Marketplace 选择 RMO，用 $FLOCK 参与募资，并按 Bonding Curve 获得该模型的 Model Token（MT）。

1

浏览 Marketplace

在 Marketplace 浏览进行中的 RMO，查看模型信息、定价档位与募资区间。

2

选择一个模型并投入 $FLOCK

选择目标 RMO，在募资期内投入 $FLOCK，按当前 Bonding Curve 价格买入 MT。

曲线越靠前，单位 MT 价格通常越低；但开盘反狙击税更高。入场需同时考虑税率与曲线位置。 回看曲线 ↑
3

实时跟踪募资进度

跟踪该 RMO 相对最低 / 最高募资目标的实时进度，判断是否可能 Graduate。

4

等待募资结果

募资结束后按结果处理：达标则模型上线，你持有 MT；未达标则 $FLOCK 本金退回（手续费不退，见 §7）。

08 — LIFECYCLE

模型上线之后发生什么

模型 Graduate 后，核心变量是真实 API 调用量。推理收入扣除成本后，进入回购销毁等链上机制。

步骤 1 / 5

使用即收入

用户通过 FLock API Platform 付费调用模型，每次推理产生收入。模型上线后，实际调用量是观察模型表现的核心指标。

使用即收入

用户通过 API Platform 付费调用模型，推理产生真实收入。上线后应重点关注实际调用量，而非仅看募资阶段数据。

回购 & 销毁

推理净收入按规则在链上回购并销毁该模型的 MT，将模型使用价值计入 MT 持有者的长期机制。

质押享折扣

持有者可质押 MT 以降低 API 调用费率。发行方也可通过补贴推理成本，吸引早期 API 使用者。

二级市场流动性：Graduate 后的 Model Token 可在 Aerodrome 等流动性枢纽交易。

09 — FAQ

常见问题

点击问题展开答案；默认折叠，方便快速扫读。

发行费用为什么是固定的 32,768 $FLOCK？

结论：这是发行方的一次性固定成本。费用覆盖模型部署、链上配置与发行流程，金额对所有 RMO 一致，便于事前测算。

「反狙击税」是干什么的？

结论：用于抑制开盘抢跑。综合税从 99% 起，每分钟下降 1%，直至回到基础费，从而提高机器人与抢跑者在开盘瞬间低成本买入的难度。 ▶ 看 99% 衰减动画

越早买入是不是一定更划算？

结论：不一定。Bonding Curve 越靠前价格越低，但开盘反狙击税更高；等待税率下降时，曲线可能已上移。需同时权衡两者，本页不构成投资建议。详见 §05.5。

如果募资失败，我会损失什么？

结论：支持者退回 $FLOCK 本金，但交易手续费不退；发行方已付的发行费也不退。可拖动 §7 滑块 对照两种结果。

Model Token 和 $FLOCK 是什么关系？

结论：两者不同。$FLOCK 是 FLock 原生代币，用于参与 RMO 募资与支付发行费用；Model Token（MT）是单个 RMA 的专属代币，一个模型对应一个 MT。

准备进入 FOMO 实操？

本手册用于理解 RMO 发行、募资参与与上线后机制。实际操作前，请以 FLock 官方文档的最新条款为准；参数、费用与规则可能随版本更新。
