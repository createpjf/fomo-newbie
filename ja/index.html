FLock Open Model Offering

FOMO 初心者ガイド
ゼロからモデル発行と参加を理解

FOMO（FLock Open Model Offering）は FLock エコシステムのモデル発行 Launchpadです。発行者は RMO（Real Model Asset Offering）を開始でき、サポーターは資金調達期間中に $FLOCK で参加し、モデルが Graduate した後は Model Token を通じて収益分配とバイバック＆バーンの仕組みに入ります。本ガイドでは、発行、参加、手数料、資金調達結果、ローンチ後の仕組みを順に説明します。

01 — WHAT

FOMO とは？

FLock はモデル訓練（AI Arena）、連合学習（FL Alliance）、API 推論（API Platform）をカバーしています。FOMO が補うのはモデル発行と資産化の入口：モデルを資金調達可能、ローンチ可能、使用量に応じて収益を生む RMA（Real Model Asset）に変えることです。

一言定義

AI モデル向け Launchpad。発行者が RMO 資金調達を開始しモデルをローンチ。モデルが実際の API 呼び出しを生むと、収益はルールに従い Model Token 関連の仕組みに入ります。

何を解決するか

モデル能力は得られますが、発行、資金調達、収益分配はしばしば不透明です。FOMO は API 利用、モデル収益、Model Token ホルダーの関係を実行可能なオンチェーンルールとして定義します。

エコシステムフロー

AI Arena（訓練）→ FL Alliance（協調）→ API Platform（推論）→ FOMO（発行と MT 仕組み）。 フロー図を展開 ↓

02 — WHO

あなたはどの役割？

FOMO は発行者とサポーターの両方に対応。まず自分の役割を確認し、対応するパス（パス A / パス B）へ進んでください。

PATH A

モデル発行者
Real Model Asset (RMA) Issuer

  • モデルをデプロイしてローンチし、API 呼び出し可能な RMA を形成したい
  • Launch ページで RMO（Real Model Asset Offering、モデル資産発行）を開始
  • 固定発行手数料（32,768 $FLOCK）および関連取引手数料の支払いが必要
  • 本ガイド「パス A」に対応
PATH B

サポーター / 参加者
Users & RMA Supporters

  • Marketplace で資金調達中の RMO を選び、$FLOCK で参加
  • Bonding Curve 価格でそのモデルの Model Token（MT）を購入
  • モデル Graduate 後は MT を保有でき、API ユーザーとしてモデルを呼び出すこともできます
  • 本ガイド「パス B」に対応

04 — PATH A · HOW

パス A：発行者がモデルを開始

RMA Issuer 向け。フローは FOMO Launch ページで完了。推論価格帯、資金調達レンジなどの主要パラメータはプラットフォームが事前設定し、発行者は随意に変更できません。

1

Launch ページへ

FOMO Launch（発行）ページを開く。RMO 開始の起点です。

2

対象モデルを選択

発行するモデルを選択。各モデルには事前設定の推論価格帯と許可された資金調達レンジが対応。

これらのパラメータはプラットフォーム固定で、価格と資金調達レンジを制約。随意変更不可。
3

自動入力パラメータを確認

モデルを選ぶと、関連数値がそのモデルに基づき自動入力されます。問題なければ確認 —— 同様に随意変更はできません。

4

（任意）初回トークン購入をバンドル

発行者は初回購入をバンドルし、Bonding Curve の初期価格帯に近づけることができます。 ↓ 曲線メカニズム

Bonding Curve は資金調達期間全体で有効。各購入が曲線価格を動かし、発行の瞬間だけの価格設定ではありません。
5

発行手数料を確認・承認

RMO を正式に開始する前に、発行者は発行関連手数料（固定発行手数料含む）を確認・承認する必要があります。手数料の詳細は §5。

6

RMO を開始し Marketplace へ

開始後、その RMO は Marketplace に表示され、公開資金調達段階に入ります。

7

資金調達終了を待つ

資金調達進捗（最小 / 最大目標に向けて）はリアルタイムで追跡。資金調達終了まで追加操作は不要（達成 / 未達は §7）。

05 — HOW MUCH

発行手数料一覧

RMO 開始には 2 種類の費用があります：発行者の一回限りの発行手数料、および参加者が購入時に支払う基本手数料と反スナイピング税。ルールを理解してから Launch または Marketplace で操作することをおすすめします。

32,768 $FLOCK
一回限り発行手数料 · One-time launch fee

RMO 開始時に発行者が支払う固定の非経常費用。モデルデプロイ、オンチェーン設定、発行フローのコストをカバー。

99% → 基本手数料
総合税 · 基本手数料 + 反スナイピング税（anti-sniping tax）

総合税は 99% から始まり、毎分 1% ずつ下がり、基本手数料に戻ります。オープン時のスナイピング（sniping）とボット掃引のコストを上げます。 ▶ 99% 減衰アニメーション

重ね合わせ視点 · 反スナイピング税の減衰

Bonding Curve の最低価格はオープン付近ですが、この時点の総合税（基本手数料 + 反スナイピング税）は 99% に達し、その後毎分 1% ずつ下がります。 因此最低曲線価格 ≠ 最低購入コスト。

曲線は上がるのみ

累計購入が増えるほど MT 単価は曲線上で右上に上昇。資金調達需要は曲線価格に直接反映されます。

早入场优势

早く参加するほど曲線位置は通常低く、同量の $FLOCK でより多くの MT を得られます（その時点の税率は差し引かれます）。発行者の初回購入バンドルは、曲線の前段に近づける意図があります。

期間中ダイナミック価格

Bonding Curve は資金調達期間全体で有効。発行の瞬間に限らず —— 各 $FLOCK 購入が価格を動かします。

最低価格 ≠ 最低コスト

曲線の前端は価格が低いですが、オープン時の反スナイピング税は高くなります。税率低下を待つ間に曲線が上がっている可能性があります —— エントリータイミングはご自身で判断してください（本ページは投資助言ではありません）。

06 — PATH B · HOW

パス B：サポーターが資金調達に参加

RMA Supporters 向け。Marketplace で RMO を選び、$FLOCK で資金調達に参加し、Bonding Curve に沿ってそのモデルの Model Token（MT）を取得します。

1

Marketplace を閲覧

Marketplace で進行中の RMO を閲覧し、モデル情報、価格帯、資金調達レンジを確認。

2

モデルを選んで $FLOCK を投入

対象 RMO を選び、資金調達期間中に $FLOCK を投入し、現在の Bonding Curve 価格で MT を購入。

曲線が前にあるほど MT 単価は通常低い。ただしオープン時の反スナイピング税は高くなります。エントリー時は税率と曲線位置の両方を考慮してください。 曲線を再確認 ↑
3

資金調達進捗をリアルタイム追跡

その RMO の最低 / 最高調達目標に対するリアルタイム進捗を追跡し、Graduate 可能性を判断。

4

資金調達結果を待つ

資金調達終了後、結果に応じて処理：達成すればモデルがローンチし MT を保有、未達なら $FLOCK 元本返還（手数料は返還されず、 §7）。

07 — OUTCOMES

資金調達結果：2 つの分岐

資金調達期間終了時、最低調達目標の達成可否がその後の経路を決めます：達成すれば Graduate してローンチ、未達なら返金フローへ。スライダーを動かして 2 つの結果を確認してください。

08 — LIFECYCLE

モデルローンチ後に何が起きるか

モデル Graduate 後、核心変数は実際の API 呼び出し量です。推論収益からコストを差し引いた後、バイバック＆バーンなどのオンチェーン仕組みに入ります。

収益メカニズム · ノードをクリックまたはホバーで停止
ステップ 1 / 5

利用即収益

ユーザーが FLock API Platform 経由で有料でモデルを呼び出し、各推論が収益を生みます。モデルローンチ後、実際の呼び出し量はモデルパフォーマンスを見る核心指標です。

デモは自動でローテーション。ホバーまたはノードクリックで一時停止して確認できます。下の 3 枚のカードは各ステップの補足です。

利用即収益

ユーザーが API Platform 経由で有料でモデルを呼び出し、推論が実収益を生みます。ローンチ後は資金調達段階のデータだけでなく、実際の呼び出し量を重点的に見てください。

バイバック & バーン

推論の純収益はルールに従いオンチェーンでそのモデルの MT をバイバック＆バーンし、モデル利用価値を MT ホルダーの長期メカニズムに組み込みます。

ステークで割引

ホルダーは MT をステークして API 呼び出し手数料を下げられます。発行者も推論コストを補助して、初期の API 利用者を引きつけられます。

二次市場流動性：Graduate 後の Model Token は Aerodrome などの流動性ハブで取引できます。

09 — FAQ

よくある質問

質問をクリックして回答を展開。デフォルト折りたたみで、素早く読めます。

発行手数料はなぜ固定の 32,768 $FLOCK なのか？

結論：発行者の一回限りの固定コストです。モデルデプロイ、オンチェーン設定、発行フローをカバーし、金額はすべての RMO で同一、事前に試算しやすくなっています。

「反スナイピング税」とは？

結論：オープン時のスナイピングを抑制するため。総合税率は 99% から始まり、毎分 1% ずつ下がり、基本手数料に戻ります。ボットやスナイパーがオープン直後に低コストで買い込む難易度を上げます。 ▶ 99% 減衰アニメーション

早く買うほど必ずお得？

結論：必ずしもそうではありません。Bonding Curve が前にあるほど価格は低いですが、オープン時の反スナイピング税は高くなります。税率が下がるのを待つ間に曲線が上がっている可能性もあります。両方を比較検討する必要があり、本ページは投資助言ではありません。詳しくは §05.5。

資金調達が失敗したら、何を失いますか？

結論：サポーターは $FLOCK 元本が返還されますが、取引手数料は返還されません。発行者が支払った発行手数料も返還されません。§7 スライダー をドラッグして 2 つの結果を対照してください。

Model Token と $FLOCK の関係は？

結論：両者は異なります。$FLOCK は FLock ネイティブトークンで、RMO 資金調達への参加と発行手数料の支払いに使います。Model Token（MT）は個別 RMA 専用のトークンで、1 モデルに 1 MT です。

FOMO の実践に進む準備はできましたか？

本ガイドは RMO 発行、資金調達への参加、ローンチ後の仕組みを理解するためのものです。実際の操作前に FLock 公式ドキュメントの最新条項を参照してください。パラメータ、手数料、ルールはバージョン更新で変わる場合があります。
